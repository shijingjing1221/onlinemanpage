NAME skb_kill_datagram - Free a datagram skbuff forcibly

SYNOPSIS int skb_kill_datagram(struct sock * sk, struct sk_buff * skb, unsigned int flags);

ARGUMENTS sk socket

skb datagram skbuff

flags MSG_ flags

DESCRIPTION This function frees a datagram skbuff that was received by skb_recv_datagram. The flags argument must match the one used for skb_recv_datagram.

If the MSG_PEEK flag is set, and the packet is still on the receive queue of the socket, it will be taken off the queue before it is freed.

This function currently only disables BH when acquiring the sk_receive_queue lock. Therefore it must not be used in a context where that lock is acquired in an IRQ context.

It returns 0 if the packet was removed by us.
