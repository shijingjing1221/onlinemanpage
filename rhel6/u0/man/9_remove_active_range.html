REMOVE_ACTIVE_RANGE(9) Memory Management in Linux REMOVE_ACTIVE_RANGE(9)

NAME remove_active_range - Shrink an existing registered range of PFNs

SYNOPSIS void remove_active_range(unsigned int nid, unsigned long start_pfn, unsigned long end_pfn);

ARGUMENTS nid The node id the range is on that should be shrunk

start_pfn The new PFN of the range

end_pfn The new PFN of the range

DESCRIPTION i386 with NUMA use alloc_remap to store a node_mem_map on a local node. The map is kept near the end physical page range that has already been registered. This function allows an arch to shrink an existing registered range.
