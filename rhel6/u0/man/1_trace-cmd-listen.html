NAME trace-cmd-listen - listen for incoming connection to record tracing.

SYNOPSIS trace-cmd listen -p port [OPTIONS]

DESCRIPTION The trace-cmd(1) listen sets up a port to listen to waiting for connections from other hosts that run trace-cmd-record(1) with the -N option. When a connection is made, and the remote host sends data, it will create a file called trace.HOST:PORT.dat. Where HOST is the name of the remote host, and PORT is the port that the remote host used to connect with.

OPTIONS -p port This option will specify the port to listen to.

-D This options causes trace-cmd listen to go into a daemon mode and run in the background.

-d dir This option specifies a directory to write the data files into.

-o filename This option overrides the default trace in the trace.HOST:PORT.dat that is created when a remote host connects.

-l filename This option writes the output messages to a log file instead of standard output.
