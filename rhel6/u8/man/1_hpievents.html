NAME hpievents - An openhpi sample application that polls for events.

SYNOPSIS hpievents [ -t -d -x ]

DESCRIPTION hpievents polls for events in an opened HPI session. User can specify wait time for the event. User can also select the order between hpi resource (resource event) discovery and hpi event subscription.

OPTIONS where: -t <value> - wait <value> seconds for event; -t SAHPI_TIMEOUT_BLOCK or BLOCK - infinite wait -d - call saHpiDiscover() after saHpiSubscribe() -x - displays eXtra debug messages

HPI APIs uniquely used in this application SaErrorT SAHPI_API saHpiSubscribe ( SAHPI_IN SaHpiSessionIdT SessionId );

SaErrorT SAHPI_API saHpiUnsubscribe ( SAHPI_IN SaHpiSessionIdT SessionId );

SaErrorT SAHPI_API saHpiEventGet ( SAHPI_IN SaHpiSessionIdT SessionId, SAHPI_IN SaHpiTimeoutT Timeout, SAHPI_OUT SaHpiEventT *Event, SAHPI_INOUT SaHpiRdrT *Rdr, SAHPI_INOUT SaHpiRptEntryT *RptEntry, SAHPI_INOUT SaHpiEvtQueueStatusT *EventQueueStatus );
