NAME

opashowmc

(Linux) Displays the Intel(R) Omni-Path Multicast groups created for the fabric along with the Intel(R) Omni-Path Host Fabric Interface (HFI) ports which are a member of each multicast group. This command can be helpful when attempting to analyze or debug True Scale multicast usage by applications or ULPs such as IPoIB.

Syntax

opashowmc [-v] [-t portsfile] [-p ports]

or
opashowmc --help

Options

--help

Produce full help text.

-v

Verbose output, show name of each member.

-t portsfile

File with list of local HFI ports used to access fabric(s) for analysis, default is /etc/sysconfig/opa/ports .

-p ports

List of local HFI ports used to access fabric(s) for analysis default is first active port.This is specified as hfi:port :

0:0 First active port in system.

0:y Port y within system.

x:0 First active port on HFI x.

x:y HFI x, port y.

The first HFI in the system is 1. The first port on an HFI is 1.

Environment Variables

The following environment variables are also used by this command:

PORTS

List of ports, used in absence of -t and -p .

PORTS_FILE

File containing list of ports, used in absence of -t and -p .
