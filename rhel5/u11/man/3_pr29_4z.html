NAME pr29_4z - check if input trigger Unicode normalization bugs

SYNOPSIS #include <pr29.h>

int pr29_4z(const uint32_t * in);

ARGUMENTS const uint32_t * in zero terminated array of Unicode code points.

DESCRIPTION Check the input to see if it may be normalized into different strings by different NFKC implementations, due to an anomaly in the NFKC speci- fications.

RETURN VALUE Returns the Pr29_rc value PR29_SUCCESS on success, and PR29_PROBLEM if the input sequence is a "problem sequence" (i.e., may be normalized into different strings by different implementations).

REPORTING BUGS Report bugs to <bug-libidn@gnu.org>.
